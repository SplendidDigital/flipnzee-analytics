# Contributing to Flipnzee Analytics

First, thank you for considering contributing to **Flipnzee Analytics**.

This project is an open-source WordPress plugin that provides transparent Google Analytics 4 and Google Search Console insights using first-party data. Every contribution helps improve the project for the WordPress community.

---

# Ways to Contribute

Contributions are welcome in many forms, including:

- Bug reports
- Feature requests
- Documentation improvements
- Performance optimizations
- UI/UX enhancements
- Security improvements
- Code contributions
- Translation suggestions
- Testing on different WordPress environments

---

# Before You Start

Before opening an issue or submitting a pull request, please:

- Search existing Issues to avoid duplicates.
- Read the README.
- Check the CHANGELOG for recent updates.
- Make sure your proposal aligns with the project's goals.

---

# Development Guidelines

Please follow these principles when contributing:

- Follow WordPress Coding Standards.
- Write clean, readable, maintainable code.
- Keep functions focused on a single responsibility.
- Escape output.
- Sanitize and validate all user input.
- Use nonces where appropriate.
- Avoid introducing breaking changes whenever possible.
- Keep backward compatibility in mind.

---

# Reporting Bugs

When reporting a bug, please include:

- WordPress version
- PHP version
- Plugin version
- Browser (if applicable)
- Steps to reproduce
- Expected behaviour
- Actual behaviour
- Screenshots (if available)

---

# Feature Requests

Feature requests should include:

- Problem being solved
- Proposed solution
- Benefits
- Possible alternatives

Please keep feature requests focused and well explained.

---

# Pull Requests

Before submitting a pull request:

- Create a separate branch.
- Test your changes thoroughly.
- Update documentation if necessary.
- Keep commits focused on a single purpose.
- Ensure no debug code or temporary files remain.

---

# Documentation

Documentation improvements are always appreciated.

Examples include:

- README improvements
- Code comments
- API documentation
- Architecture diagrams
- Screenshots
- Tutorials

---

# Security

If you discover a security vulnerability, please do **not** disclose it publicly.

Instead, contact the project maintainer privately so the issue can be investigated and resolved responsibly.

---

# Coding Standards

The project follows:

- WordPress Coding Standards
- PHP Best Practices
- Secure Coding Principles
- Object-Oriented Design where appropriate
