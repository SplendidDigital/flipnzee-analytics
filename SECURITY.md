# Security Policy

Thank you for helping keep **Flipnzee Analytics** secure.

The security of this project is taken seriously. If you discover a security vulnerability, please report it responsibly so it can be investigated and fixed before public disclosure.

---

# Supported Versions

The following versions currently receive security updates.

| Version | Supported |
|----------|-----------|
| 2.9.x | ✅ Yes |
| 2.8.x | ❌ No |
| Older versions | ❌ No |

---

# Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub Issues.

Instead, contact the project maintainer privately with as much information as possible, including:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Proof of concept (if available)
- Suggested mitigation (optional)

Please include:

- WordPress version
- PHP version
- Plugin version
- Browser (if applicable)

---

# Responsible Disclosure

After receiving a report, the following process will generally be followed:

1. Confirm receipt of the report.
2. Investigate the issue.
3. Develop and test a fix.
4. Release a patched version.
5. Publicly disclose the issue after users have had time to update.

We appreciate responsible disclosure and ask that vulnerabilities are not publicly disclosed until a fix has been released.

---

# Security Best Practices

Flipnzee Analytics follows WordPress security best practices wherever possible, including:

- Input sanitization
- Output escaping
- Nonce verification
- Capability checks
- Prepared SQL statements
- Secure API authentication
- Principle of least privilege

Security improvements are continuously reviewed as the project evolves.

---

# Supported Environment

The plugin is intended to run on supported versions of:

- WordPress
- PHP
- Google Analytics 4 Data API
- Google Search Console API

Keeping your WordPress installation, PHP version, themes, and plugins up to date is strongly recommended.

---

# Third-Party Services

Flipnzee Analytics integrates with:

- Google Analytics 4
- Google Search Console

Users are responsible for securing their Google accounts and OAuth credentials.
